我们的网站某些操作需要用户登录才能进行，如收藏、评论等。还有一些操作只有管理员可以进行，需要拒绝普通用户的这些操作，因此需要鉴别用户的身份。

# 一、关于JWT

JSON Web Token（JWT）是一种开放标准（RFC 7519），用于在网络应用间安全地传输信息。它以紧凑且自包含的方式，通过 JSON 对象在各方之间传递经过验证的信息。

JWT 由三部分组成，用 `.` 分隔：

**Header**（头部）：包含算法（如 HMAC SHA256 或 RSA）和令牌类型（固定为 `JWT`）。

```json
{
  "alg": "HS256",
  "typ": "JWT"
}
```

**Payload**（负载）：携带声明（如用户身份、权限、有效期等），分为三类：

- **Registered claims**（预定义字段，如 `exp` 过期时间、`iss` 签发者）。
- **Public claims**（公开自定义字段，需避免冲突）。
- **Private claims**（私有字段，双方协商）。

```json
{
  "sub": "1234567890",
  "name": "John Doe",
  "admin": true
}
```

- **Signature**（签名）：对头部和负载的**签名**，防止数据篡改。

需要注意的是，JWT只是将一些信息转换成了base64url,看起来是加密了，实际上只是换了一种编码，反解一下可以直接转换得到原来的Token信息，因此**不适合直接在JWT中存储用户名、密码、手机号等一些隐私数据**：

## 二、配置JWT

在 `pom.xml` 中的 `<dependencies>` 中，添加JWT的依赖：

```xml
<dependency>
    <groupId>com.auth0</groupId>
    <artifactId>java-jwt</artifactId>
    <version>4.3.0</version>
</dependency>
```
