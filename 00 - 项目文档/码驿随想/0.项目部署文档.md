# 本地开发与上传服务器步骤

## 1.本地开发环境下：配置修改

- mysx_back/mysx_back/**settings.py**:
	- **line 23**：is_deploy = False

- mysx_front/src/services/**api.js**:

```bash
# 运行前端开发服务器
cd mysx_front
npm run dev

# 运行后端开发服务器
cd mysx_back

```

## 2.打包与服务器文件部署


### 预备工作

- 停止本地前后端服务器，备份数据库
- 恢复上面两个文件，后端选用部署模式，前端请求目标改为api路径

### 前端打包上传

0. 停止前端开发服务器，运行命令 `npm run build`
1. 登陆宝塔，停止前端站点
2. dist.zip上传解压到/www/wwwroot/mysx-vue
3. 开启前端站点
4. 开启代理，登录cloudflare控制台，域 - 缓存 -配置 - 清除缓存 - 清除所有内容

### 后端打包上传

0. 停止开发服务器，更新依赖：
```bash
# 更新requirements.txt
pip freeze > requirements.txt

# 检查去掉pywin32==311之类的win下Python环境
```

1. 压缩mysx_back文件夹为zip压缩包
2. 登陆宝塔，停止后端站点
3. mysx_back.zip上传解压到/www/wwwroot/mysx_django
```bash
# 进入mysx_back文件夹，激活虚拟环境并更新依赖
source .venv/bin/activate
pip install -r requirements.txt --upgrade
```

4. 运行后端网站
5. 开启代理，登录cloudflare控制台，域 - 缓存 -配置 - 清除缓存 - 清除所有内容
