# 一、汇编语言的特点

- 速度快，代码量小
- 在计算机中位于底层，直接控制硬件，了解后可以更好地使用高级语言
- 多用于加/解密、病毒防治、信息安全/逆向工程等领域

# 二、冯·诺依曼计算机组成

冯·诺依曼体系的核心就是 **“存储程序，顺序执行”** —— 程序事先存入存储器，然后CPU按地址顺序一条条取出执行。这一结构奠定了现代通用计算机的基础。

**主机**：包含**控制器、运算器、存储器**  
**I/O**：输入输出设备
![](20260507140750102.jpg)

- **ALU（算术逻辑单元）**：可做加法、移位两类操作
- **寄存器**：指CPU中的寄存器，存取速度快于主存储器

# 三、汇编语言的编译过程

```
汇编源程序 --(汇编)--> 目标程序 --(连接)--> 可执行程序
*.asm                  *.obj                *.exe
```
与C/C++程序的编译过程类似，也是先编译成.obj文件，然后再连接得到.exe文件。

# 四、8086 的功能结构

![](20260507140933708.jpg)
## 1. 总线接口单元（BIU）

主要功能：负责数据传输
组成：6 字节指令队列、指令指针（IP）、段寄存器（CS、DS、SS、ES）、地址加法器、总线控制逻辑等

## 2.执行单元（EU）

主要功能：负责执行程序中的指令
组成：算术逻辑单元 ALU、数据寄存器、地址寄存器、标志寄存器、指令译码的 EU 控制逻辑等

## 3.指令执行的两个主要阶段

1. **取指**：从主存中取出指令代码进入 CPU
2. **执行**：将指令代码翻译成它代表的功能（译码）

# 五、8086 的寄存器

## 1.通用寄存器

在处理器中数量较多，使用频率较高，具有多种用途，如存储指令需要的数据、地址等。

## 2.专用寄存器

只用于特定用途，如IP（指令指针）用于寄存将要执行指令的主存地址，FLAGS（标志寄存器）用于存储指令执行的辅助信息。

# 六、8个16位通用寄存器

## 1.寄存器概览

| 寄存器 | 名称                   | 用途                     |
| --- | -------------------- | ---------------------- |
| AX  | 累加器（Add）             | 使用频度高，用于算逻运算及外设通信      |
| BX  | 基址寄存器（Base）          | 常用于存放存储器地址，方便指向变量/数组   |
| CX  | 计数器（Count）           | 作为循环操作等指令中的计数器         |
| DX  | 数据寄存器（Data）          | 存放数据，在 I/O 指令中存放外设端口地址 |
| SI  | 源变址寄存器（Source）       | 指向字符串或数组的源操作数          |
| DI  | 目的变址寄存器（Destination） | 指向字符串或数组的目的操作数         |
| BP  | 基址指针寄存器（Base）        | 默认指向堆栈区                |
| SP  | 堆栈指针寄存器（Stack）       | 专用于指向栈顶区域顶部的数据         |
## 2.数据寄存器

AX、BX、CX、DX是四个**数据寄存器**，每个可分成高字节（H）和低字节（L）两部分，得到 8 个 8 位通用寄存器：

| 16 位 | 高字节 (15-8) | 低字节 (7-0) |
| ---- | ---------- | --------- |
| AX   | AH         | AL        |
| BX   | BH         | BL        |
| CX   | CH         | CL        |
| DX   | DH         | DL        |
