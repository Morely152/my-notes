## 1. 项目概述
- **项目名称**：**浮境（Floating Dream）**
- **核心理念**：利用 LLM 创作与 TTS 技术，为现代失眠人群提供高度个性化、低蓝光刺激的助眠故事与氛围环境。
- **产品矩阵**：包含 Flutter 移动客户端（核心体验）、FastAPI 后端服务（逻辑与对账）及 Web 管理端（内容审核与数据监控）。
---

## 2. 核心功能需求

### 2.1 氛围与故事双轨播放系统

为满足用户对“环境背景+人声朗读”的混音需求，系统必须支持双路音频独立并行处理。

- **技术实现**：
    - **独立实例**：构建两套相互隔离的播放器线程（Player A & Player B），防止资源竞争导致音效卡顿。
    - **氛围播放**：采用三列卡片布局，支持雨声、篝火等素材的无缝循环播放。
    - **故事播放**：支持流式加载，首字响应（TTFB）需控制在 3 秒内。

### 2.2 “入眠遥控器”控制中心 (关键细节)

该页面是用户入睡前的最后操作界面，需保证极简且高效。

- **双栏独立控制**：
    - **左栏（氛围音）**：点击中心区域切换播放/暂停，垂直滑动调节环境音量。
    - **右栏（故事语音）**：点击中心区域切换播放/暂停，垂直滑动调节朗读音量。

- **总控与淡出**：顶部 20% 区域设置定时关机，倒计时结束时两路音频需执行 5-10 秒的平滑淡出，严禁突然切断声音。

### 2.3 AI 故事生成与广场

- **自定义引导**：通过对话式交互选择关键词（如：森林、星际、科普）引导 LLM 生成 10-60min 的长文本。
- **浮境广场**：采用瀑布流布局分发优质 UCG 内容，支持 15 秒试听，满意后获取完整音频（鼓励减少Token开支，广场音频不收取积分）。

## 3. 专项处理注意事项

### 3.1 账务处理：准确性与安全性 (关键细节)

- **事务一致性**：所有积分充值与 Token 消耗必须接入数据库事务。执行“扣费”动作必须在后端确认 AI 接口请求成功后完成，防止用户积分流失却未获得内容。
- **对账系统**：系统每日自动比对支付网关记录、积分账户余额与日志流转，任何超过 1 分钱的误差需触发系统警报。
- **流控防御**：针对恶意刷接口行为设置单用户日生成上限，保障 API 成本在预算范围内。

AI 生成成本（Token）是项目最大的开支，必须平衡用户爽感与财务安全。

- **Token 缓冲池（预生成机制）**：
    - **建议**：对于热门关键词（如：森林、雨天、星际），后端可以预先生成一批高质量故事文本和音频。当用户选择大众化主题时，优先下发缓存内容，仅需扣除极低积分甚至免费，只有真正的“深度定制”才消耗高额 Token。
        
- **“重听”激励政策**：
    - **建议**：用户生成的旧故事再次收听时不应重复计费。鼓励用户将好故事分享到广场，如果被他人收藏或付费收听，原创作者应获得积分返利，从而激励 UCG 内容产出，降低平台整体的 AI 生成成本。

### 3.2 UI/UX：助眠核心主题 (关键细节)

- **色调方案**：全屏采用**深暖色调**（Dark Mode）。底色使用深灰或墨黑，文字与图标使用低饱和度的琥珀色或暖白，严禁使用任何高亮度冷蓝光。
- **交互节奏**：所有转场和按钮反馈需具备“呼吸感”，使用线性淡入淡出动画，严禁闪屏或生硬的即时跳变。
- **操作路径**：确保用户在睡意朦胧时，从打开 App 到开启播放的操作路径不超过 3 步。

### 3.3 安全与审核

- **双重过滤**：生成文本需先经过阿里云内容安全 AI 审核，剔除恐怖、色情或令人兴奋的内容；广场内容由人工定期抽检。

## 4. 技术与非功能指标

- **稳定性**：支持 8 轨混音时 CPU 占用率低于 15%，音频引擎需具备系统级白名单保活能力，防止后台被误杀。
- **日志监控**：记录完整的前端 API 调用日志与后端账务流水，确保运营故障可回溯。

### 4.1 系统稳定性：针对“断点”的防御

睡眠过程中的任何软件故障（如爆音、断流）都会让用户瞬间清醒，导致产品流失。

- **混合缓存策略**：
    - **建议**：强制要求在播放前预加载至少 2 分钟的音频 Buffer。对于氛围音，必须在本地存储 OGG/WAV 循环片断，严禁在播放中途进行网络拉取，防止因信号波动导致的音频卡顿。

- **后台保活“白名单”引导**：
    - **建议**：在新手引导阶段，针对安卓用户增加一个“电池优化白名单”设置入口。如果 App 在后台被系统强杀，会导致用户惊醒并产生极差评价。

### 4.2 视觉与心理：强化“安全感”

- **动态呼吸灯特效**：
    - **建议**：入眠页面的 UI 背景可以配合用户的呼吸频率进行极缓慢的明暗变化。这种视觉暗示能引导用户调整呼吸节奏，加速入眠。
