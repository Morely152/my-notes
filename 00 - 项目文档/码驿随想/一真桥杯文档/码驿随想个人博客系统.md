# 一、竞品分析：现有博客的体验问题

- **公众博客平台（CSDN、博客园、稀土掘金等）**：
	- 流量较多，但是文章内容受限、推流机制不透明，广告信息多；
	- 功能完整，但是风格统一、结构传统，较难实现自定义主题；
	- 附件上传麻烦，本地图片需要逐张插入，部分平台对MarkDown支持不足，格式调整困难。
- **静态博客网页（Hexo/Hugo + Github Pages托管）**：
	- 部署方便，但是纯展示型，无后端服务，管理不方便。
# 二、解决方案：为什么选择自己搭博客

- 采用Vue3+Django前后端分离的经典架构，技术成熟，便于实现自定义的现代化、美观简约的界面风格；
- 通过引入md-editor-v3工具，完美实现对MarkDown文章的编辑和渲染；
- 重点优化了本地图片上传的流程，可实现文章附图一键批量上传；
- 自建博客的可拓展性高，在已有的用户模块、文章模块上引入AI助手、RAG等新技术，是应用新技术的良好实战平台；
# 三、系统架构分析

- 前端：Vue3
	- 软件生态全面，组件工具丰富；
	- 基于前端三剑客，上手难度低，开发效率高；
- Node包：
	- md-editor-v3：提供MarkDown编辑器和渲染组件，完美解决文章的编辑与展示需求；
	- Element Plus：兼容Vue3的开源组件库，采用开箱即用的模块化设计，功能齐全；
- 后端：Django
	- 常用工具齐全，安全防护全面；
	- REST framework架构提升接口开发便捷性；
- 数据库：MySQL
	- 企业级解决方案，应用广阔；
	- 开源免费，性能强大，稳定性好；

# 五、技术创新亮点

## 1. 前端深浅色主题配置、响应式布局应用

- 适配深浅色双主题，用户体验好
- 规范化UI设计，整体页面视觉效果一致
- 基于CSS媒体查询，实现PC/平板/手机三端响应式布局

## 2. 文章关系图谱展示

- 后端根据文章专栏结构，将文章关系编制成JSON数据
- 前端将JSON数据渲染成Echarts力导向图

# 3.文章Markdown编辑、图片批量上传

- 在管理端上传时，可以对文章进行修改
- 支持批量上传文章中的图片附件，将本地路径替换为网络链接

# 六、后续升级方向

- 接入AI智能体，将文章数据转化为知识库，实现RAG（增强检索生成）
