## 2.打包与服务器文件部署


### 预备工作

- 停止本地前后端服务器，备份数据库
- 恢复上面两个文件，后端选用部署模式，前端请求目标改为api路径

### 前端打包上传

0. 停止前端开发服务器，运行命令 `npm run build`
1. 登陆宝塔，停止前端站点
2. dist.zip上传解压到/www/wwwroot/mysx-vue
3. 开启前端站点
4. 开启代理，登录cloudflare控制台，域 - 缓存 -配置 - 清除缓存 - 清除所有内容

### 后端打包上传

0. 停止开发服务器，更新依赖：
```bash
# 更新requirements.txt
pip freeze > requirements.txt

# 检查去掉pywin32==311之类的win下Python环境
```

1. 压缩mysx_back文件夹为zip压缩包
2. 登陆宝塔，停止后端站点
3. mysx_back.zip上传解压到/www/wwwroot/mysx_django
```bash
# 进入mysx_back文件夹，激活虚拟环境并更新依赖
cd mysx_back
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt --upgrade
```

4. 修改`.env`中的开发环境配置
5. 运行后端网站
6. 开启代理，登录cloudflare控制台，域 - 缓存 -配置 - 清除缓存 - 清除所有内容

后端运行命令

```bash
cd /www/wwwroot/mysx-django/mysx_back
source venv/bin/activate
python --version

手动更新可视化数据

常用管理命令：

  # 查看是否在运行
  ps -ef | grep gunicorn

  # 查看端口

  常用命令：

  # 启动

  # 重启
  systemctl restart mysx-back

  # 查看状态
  systemctl status mysx-back --no-pager

  journalctl -u mysx-back -f
```
