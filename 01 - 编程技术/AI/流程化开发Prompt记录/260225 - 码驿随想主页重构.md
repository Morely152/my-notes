### 2. 增强版 Spec-kit 指令集（补充细节处理）

为了保证重构后的效果，我们需要在指令中加入关于**纹理、光影、动态曲线**的细节。

#### ① /speckit.constitution: 视觉与交互灵魂

``` markdown
设定重构的“仪式感”底层标准：
 
 1. **物理质感**：在全局 `body` 上层覆盖一层透明度为 3% 的 **噪点纹理 (Noise Texture)**，模拟纸张纤维感。
     
 2. **动态节奏**：所有翻页过渡必须符合 `cubic-bezier(0.65, 0, 0.35, 1)`（慢速启动-中间加速-平滑停止），营造沉稳的叙事感。
     
 3. **色彩空间**：严格执行 @ 中的UI规范，需要适配深浅色主题。所有颜色过渡时间设定为 `0.6s`，且在深浅切换时，背景色的色温需有微妙变化。
```
#### ② /speckit.specify: 场景细节深度描述

```markdown
任务内容：重构博客主页的UI @  ，将整个网页分成以下三页全屏场景，对应的页面结构和杨式布局参考 @  。
详细定义三个全屏场景的细节表现：
 1. **Section 1 (Hero)**：
     - **每日一言**：使用 `italic` 倾斜处理。文字下方添加一个极淡的 **“墨迹渐变”** 装饰（Opacity 0.1）。
     - **滚动指示器**：底部居中放置一条垂直细线，高度由 0 伸长至 40px 再缩回的无限循环动画。
 2. **Section 2 (Posts)**：
     - **非对称布局**：文章列表与左侧标题形成 1:2 的黄金比例分割。
     - **悬停细节**：当鼠标悬停在文章标题上时，标题颜色由炭黑变为陶土红，且背景出现极细微的垂直位移（2px）。
 3. **Section 3 (Gallery)**：
     - **照片墙**：禁止网格对齐。利用绝对定位，结合 `rotate`（-3deg 到 3deg 随机）和不同的 `z-index` 营造“散落”感。
     - **视差效果**：利用 GSAP 实现每张照片随滚动速度不同的微弱位移。
```
#### ③ /speckit.plan: 核心技术栈方案

```markdown
定义技术实现逻辑（基于 Vue 3 + Tailwind）：
 
 1. **对齐逻辑**：对 `main` 容器应用 `scroll-snap-type: y mandatory`；对每个 `section` 应用 `scroll-snap-align: start`。
     
 2. **动画引擎**：引入 `gsap` 和 `ScrollTrigger`。
     
 3. **状态驱动**：使用 `Intersection Observer` 监听当前屏索引。当 `activeSection === 1` 时，触发文章列表的逐行交错淡入（Stagger Animation）。
     
 4. **照片墙实现**：照片使用 `object-fit: cover` 确保比例，外层包裹 `div` 设置 `padding` 模拟白边相纸。

```
